# 视觉设备选型与架构指南

## 1. 简介

在智能物流及工程实践赛事中，视觉处理系统充当机器人的环境感知枢纽。视觉模块的核心功能集中于环境特征提取与目标坐标解算。其实际应用严格划分为两大场景：**车辆位置定位**与**物料定位**。

车辆位置定位旨在解决机器人在全局或局部地图中的自主导航问题。视觉系统通过识别地面引导线、交叉路口标识或特定的空间信标（如墙面 Apriltag 阵列），解算出车体当前的位姿（X、Y坐标及偏航角）。此过程要求视觉设备具备高帧率采集能力，确保底盘控制闭环的实时性，避免车辆在高速行进中偏离既定轨迹。

物料定位关注于目标物品的精准识别与空间坐标提取。物流环节中的物料通常表现为带有颜色编码、二维码标签或特定几何形状的待抓取块。视觉设备需在复杂的环境光线干扰下，准确分割出物料轮廓，定位其形心坐标，并将数据解算为机械臂可识别的三维空间抓取点。此场景对图像处理算法的鲁棒性及特征提取精度提出了较高要求。

## 2. 设备介绍

当前机器视觉边缘设备的发展呈现出传统图像处理与深度学习加速并行的趋势。在保证模块化开发与系统扩容便利性的前提下，选用合适的计算平台是项目成功的关键。

**OpenMV**
OpenMV 定位于低功耗、轻量级的嵌入式机器视觉平台。其硬件核心通常采用高性能 ARM Cortex 微控制器，软件层面高度集成 MicroPython 栈，极大降低了机器视觉应用的开发门槛。
- **应用定位**：系统设计高度模块化，原生提供的图像处理算法库（斑点追踪、线段检测、条码识别）极为完善。在智能物流场景中，极其适合承担场地特征明确的车辆位置定位（基础巡线）以及标准色块物料定位任务。设备运行逻辑透明，无复杂底层操作系统导致的意外中断。资源开销极小，硬件集成难度低。在不涉及复杂语义环境时，是稳定性和开发效率的最优解。

**K230 (CanMV)**
K230 是搭载独立神经网络处理单元 (KPU) 的边缘端 AI 计算设备。其处理器架构采用双核 RISC-V 异构设计，专为高维矩阵运算与深度推理提供硬件级加速。
- **应用定位**：填补了传统微控制器在复杂图像特征识别上的算力空白。其应用定位指向高难度的物料定位场景，如杂乱堆叠物料的分类、手写文本的数字识别以及非标准障碍物检测。该设备支持 YOLO 等主流神经网络模型的本地部署，赋予视觉模块空间泛化与语义特征学习的能力。虽然底层 RT-Smart 与 Linux 的双系统交互机制相对复杂，但借助 CanMV 封装的 Python 接口，开发者依然能够平滑扩展多传感器融合业务模块。

## 3. 设备的比较

硬件架构与生态构成的实质性差异决定了各自的优势区间。下表从工程实践角度详列其核心对比指标：

| 对比维度 | OpenMV (以 H7 Plus 为例) | K230 (CanMV开发板) |
| :--- | :--- | :--- |
| **核心运算单元** | ARM Cortex-M7 (单核物理 CPU) | 双核 RISC-V + 第三代智能处理单元 (KPU) |
| **主频性能** | 约 480 MHz | 1.6 GHz大核 / 800 MHz小核 |
| **内存配置** | 32MB SDRAM | 512MB LPDDR3 (容纳庞大模型参数) |
| **AI 算力级别** | 依赖纯 CPU 算力，难以运行深层网络 | 硬件级支持 INT8/INT16 并行推理计算 |
| **操作系统层** | 裸机/实时操作系统微内核 | 大核 Linux + 小核 RT-Smart 异构架构 |
| **图像帧率表现** | 色块追踪等基础任务可达 60fps+ | 运行 YOLO 等深度框架对象检测可达 30fps+ |
| **通讯与适配** | 支持 UART/I2C/SPI，逻辑电平 3.3V | 支持 UART/I2C/SPI，逻辑电平 3.3V |
| **模型部署生态** | TFLite Micro、Edge Impulse 闭环 | 支持 PyTorch/ONNX 经 nncase 编译生成 kmodel部署 |
