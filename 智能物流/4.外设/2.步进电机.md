## 2. 步进电机 & 5. 电机

> **一句话概括**：让小车学会"走路"——控制轮子转动的电机系统

### 2.1 想象一下

你在骑自行车时：
- **拧油门**（给电机供电）→ 车轮转动
- **看码表**（编码器）→ 知道车速是多少
- **大脑根据速度调整**（PID 控制器）→ 保持匀速前进

这就是电机控制的三个核心：**驱动** → **反馈** → **控制**

---

### 2.2 电机是怎么转起来的？

#### 硬件连接

每个轮子由一个 **H桥** 控制，就像一个双向开关：

```
   +Battery+
       |
   +---+---+
   |       |
   | H桥   |
   |       |
   +---+---+
     |   |
     |   +---> 轮子转动方向 A
     |
     +-----> 轮子转动方向 B
```

#### 代码怎么写？

```c
// 这就相当于"拧油门"
void MotorA_SetSpeed(int speed)
{
    if (speed >= 0)  // 正转
    {
        __HAL_TIM_SET_COMPARE(&htim10, TIM_CHANNEL_1, speed);  // 供电
        __HAL_TIM_SET_COMPARE(&htim11, TIM_CHANNEL_1, 0);       // 断开
    }
    else  // 反转
    {
        __HAL_TIM_SET_COMPARE(&htim10, TIM_CHANNEL_1, 0);
        __HAL_TIM_SET_COMPARE(&htim11, TIM_CHANNEL_1, -speed);
    }
}
```

**通俗解释**：
- PWM 就像你快速开关水龙头，水流大小 = 开的时间比例
- STM32 产生 PWM 信号控制 H 桥，从而控制电机转速和方向

### 2.3 怎么知道车轮转多快？

#### 编码器原理

想象一下自行车的**发电机灯**：
- 轮子转一圈，灯闪 N 次
- **数闪烁次数** → 知道转了多少圈
- **除以时间** → 知道转速

STM32 的编码器模式更高级：能识别转动的**方向**（正转还是反转）

#### 代码实现

```c
void Encoder_Update(Encoder_t *encoder)
{
    // 1. 读取"计数器"——车轮转了多少个刻度
    int32_t current_counter = __HAL_TIM_GET_COUNTER(encoder->htim);
    
    // 2. 算一下这一小段时间转了多少
    int32_t delta = current_counter - encoder->Last_counter;
    
    // 3. 处理"溢出"——计数器从 65535 回到 0 的情况
    if (delta > 32767) delta -= 65536;
    else if (delta < -32768) delta += 65536;
    
    // 4. 更新速度和累计路程
    encoder->Total_counter += delta;  // 总共转了多少
    encoder->Velocity = delta * VELOCITY_COEFFICIENT;  // 速度 = 脉冲数 × 系数
}
```

### 2.4 PID 控制器——让车开得更稳

#### 为什么要 PID？

想象你骑自行车下坡：
- **只有 P（比例）**：你会一直加速，刹不住 → **超调**
- **只有 I（积分）**：反应太慢，坡度变了才想起来加速 → **滞后**
- **只有 D（微分）**：只关心速度变化率，不关心实际速度 → **不稳**

**PID = 人的大脑**：综合考虑"现在速度对不对"、"过去累积误差"、"将来会怎样"

#### PID 代码

```c
float PID_Calculate(PID_Controller_t *pid, float target, float measured)
{
    pid->Error = target - measured;  // 现在的误差
    
    // P：现在的问题
    float p_term = pid->Kp * (pid->Error - pid->PrevError);
    
    // I：过去累积的问题
    float i_term = pid->Ki * pid->Error;
    
    // D：将来可能的问题（趋势）
    float d_term = pid->Kd * (pid->Error - 2*pid->PrevError + pid->PrevPrevError);
    
    pid->Output += p_term + i_term + d_term;
    
    // 记录历史
    pid->PrevPrevError = pid->PrevError;
    pid->PrevError = pid->Error;
    
    return pid->Output;
}
```

### 2.5 定时中断——让控制有条不紊

#### 为什么需要定时中断？

想象你在同时做两件事：
- **主循环**：看导航、听音乐、跟乘客聊天
- **定时任务**（每 10ms）：看码表、调整油门

如果全挤在一起会乱套！所以要**分工合作**。

#### 代码

```c
// TIM6 每 10ms 触发一次中断
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM6)  // 10ms 到期了！
    {
        // 1. 更新编码器——看看轮子转多快
        Encoders_All_Update();
        
        // 2. 计算 PID——根据速度调整油门
        Motor_PID_Update_All();
        
        // 3. 转向控制——保持直线行驶
        Yaw_PID_Update();
    }
}
```

### 2.6 常见问题与应对

| 问题 | 就像... | 解决方法 |
|------|---------|----------|
| 电机抖动 | 骑自行车一卡一卡的 | 调低 P 参数，或增加滤波 |
| 停车过冲 | 刹车刹不住，滑过去 | 增加 D 参数，或用斜坡减速 |
| 转向跑偏 | 自行车龙头歪了 | 检查编码器方向设置是否一致 |
| 编码器跳数 | 码表乱跳 | 检查接线是否松动，加滤波 |
