## 2. 步进电机 & 5. 电机

> **一句话概括**：让小车学会"走路"——控制轮子转动的电机系统

### 2.1 先理解：电机是什么？

想象一下你小时候玩的**四驱车**：
- 装上电池 → 轮子转 → 车跑
- 电池正接 → 车轮正转
- 电池反接 → 车轮反转

这就是**直流电机**的基本原理。但比赛用的小车可不能这样简单粗暴，我们需要：
1. **调速**：不能一直全速跑，要根据情况加速减速
2. **反馈**：要知道当前跑了多快
3. **控制**：根据目标速度自动调整

### 2.2 电机是怎么转起来的？

#### 2.2.1 H桥驱动——正反转的奥秘

想象你用手推箱子：
- 左手推 → 往右移
- 右手推 → 往左移
- 同时推 → 刹车

**H桥**就是这个原理：

```
       +电池正极
           |
       +---+---+
       |       |
   A   |       |   B
       |       |
       +---+---+
           | 
           +---→ 轮子
           |
       +---+---+
       |       |
   C   |       |   D  
       |       |
       +---+---+
           |
       +电池负极
```

**控制逻辑**：
- A+D 通电 → 正转
- B+C 通电 → 反转
- A+B 或 C+D → 刹车
- 全断开 → 滑行

#### 2.2.2 PWM 调速——模拟电压

问题：STM32 只能输出 0V 或 3.3V，怎么控制电机转速？

**解决：PWM（脉宽调制）**

这就好像**快速开关水龙头**：
- 开 1 秒、关 1 秒 → 平均水流 50%
- 开 0.5 秒、关 1.5 秒 → 平均水流 25%
- 开关速度足够快 → 电机以为收到了连续的可变电压

```c
// STM32 代码
void MotorA_SetSpeed(int speed)
{
    if (speed >= 0)  // 正转
    {
        // TIM10 输出 PWM
        __HAL_TIM_SET_COMPARE(&htim10, TIM_CHANNEL_1, speed);
        // TIM11 输出 0（断开）
        __HAL_TIM_SET_COMPARE(&htim11, TIM_CHANNEL_1, 0);
    }
    else  // 反转
    {
        __HAL_TIM_SET_COMPARE(&htim10, TIM_CHANNEL_1, 0);
        __HAL_TIM_SET_COMPARE(&htim11, TIM_CHANNEL_1, -speed);
    }
}
```

### 2.3 怎么知道车轮转多快？

#### 2.3.1 编码器原理

想象自行车发电机灯：
- 轮子转一圈 → 磁铁经过线圈 N 次 → 产生 N 个脉冲
- **数脉冲** → 知道转了多少圈
- **除以时间** → 知道转速

但自行车灯只能知道转了多少，**不知道方向**！

#### 2.3.2 正交编码器——能识别方向

**正交编码器**有两根线 A 和 B：
- A 超前 B 90° → 正转
- B 超前 A 90° → 反转

就像两个人举牌子：
- A 先举 → 往前走
- B 先举 → 往后走

STM32 的**编码器模式**自动处理这个逻辑：

```c
void Encoder_Update(Encoder_t *encoder)
{
    // 1. 读取计数器——车轮转了多少个"刻度"
    int32_t current = __HAL_TIM_GET_COUNTER(encoder->htim);
    
    // 2. 计算这一小段时间转了多少
    int32_t delta = current - encoder->Last_counter;
    
    // 3. 处理 16 位计数器溢出
    // 想象计数器从 65535 跳回 0，就像钟表从 23:59 跳到 00:00
    if (delta > 32767) delta -= 65536;
    else if (delta < -32768) delta += 65536;
    
    // 4. 左轮要翻转符号（机械安装方向相反）
    if (encoder->htim->Instance == TIM3 || encoder->htim->Instance == TIM4)
    {
        delta = -delta;
    }
    
    // 5. 更新数据
    encoder->Total_counter += delta;  // 总共转了多少
    encoder->Velocity = delta * VELOCITY_COEFFICIENT;  // 转速
}
```

### 2.4 PID 控制器——让车开得更稳

#### 2.4.1 为什么要 PID？

想象你骑自行车下坡：
- **不控制**：越来越快 → 失控 → 危险！
- **只用 P（比例）**：发现快了 → 刹车 → 太快又加速 → 来回震荡
- **只用 I（积分）**：累积之前的误差 → 反应太慢
- **只用 D（微分）**：只关心变化趋势 → 稳定但有静差

**PID = 综合考虑现在 + 过去 + 将来**

#### 2.4.2 三个参数的含义

| 参数 | 名字 | 比喻 | 调整效果 |
|------|------|------|----------|
| Kp | 比例 | 现在犯的错有多大 | 大→反应快但震荡，小→稳但慢 |
| Ki | 积分 | 过去累积错了多少 | 消除静差，但可能超调 |
| Kd | 微分 | 将来可能错多少 | 阻尼震荡，但会放大噪声 |

#### 2.4.3 PID 代码实现

```c
float PID_Calculate(PID_Controller_t *pid, float target, float measured)
{
    // 现在的问题
    pid->Error = target - measured;
    
    // P：现在的问题有多大
    float p_term = pid->Kp * (pid->Error - pid->PrevError);
    
    // I：过去累积的问题
    float i_term = pid->Ki * pid->Error;
    
    // D：将来看起来会怎样（趋势）
    float d_term = pid->Kd * (pid->Error - 2*pid->PrevError + pid->PrevPrevError);
    
    // 输出是变化量
    pid->Output += p_term + i_term + d_term;
    
    // 记录历史
    pid->PrevPrevError = pid->PrevError;
    pid->PrevError = pid->Error;
    
    return pid->Output;
}
```

### 2.5 定时中断——让控制有条不紊

#### 2.5.1 为什么需要中断？

想象你同时做两件事：
- **主循环**：看导航、听音乐、跟乘客聊天（优先级低）
- **定时任务**（每 10ms）：看码表、调整油门（优先级高）

如果全挤在一起：
- 你正在听音乐，看码表忘了
- 你正在看码表，音乐停了

所以要**分工合作**：
- 主循环做长时间的事（UI显示、串口通信）
- 中断做紧急的事（PID控制）

#### 2.5.2 TIM6 中断

```c
// TIM6 每 10ms 触发一次（100Hz）
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM6)
    {
        // 1. 先看"码表"——更新编码器
        Encoders_All_Update();
        
        // 2. 根据速度调整"油门"——PID计算
        Motor_PID_Update_All();
        
        // 3. 保持"方向"——转向环
        Yaw_PID_Update();
    }
}
```

### 2.6 运动模型——四轮麦克纳姆轮

#### 2.6.1 麦克纳姆轮的特点

普通轮子只能**前进后退**，麦克纳姆轮可以**横着走**！

原理：轮子边缘有斜向滚子

```
  ────↘  ────↙
  ──↘    ↙──
  ↘      ↙
```

#### 2.6.2 运动控制

```c
// 四轮分别控制
void Car_Move(int speed_left, int speed_right)
{
    // 左侧两个轮子
    MotorB_SetSpeed(speed_left);   // 左前
    MotorC_SetSpeed(speed_left);   // 左后
    
    // 右侧两个轮子
    MotorA_SetSpeed(speed_right);  // 右前
    MotorD_SetSpeed(speed_right);   // 右后
}

// 前进
void Car_Go(int speed)
{
    Car_Move(-speed, -speed);  // 负值=前进（机械安装决定）
}

// 左转
void Car_TurnLeft(int speed)
{
    Car_Move(speed, -speed);  // 左后退，右前进 → 原地左转
}
```

### 2.7 常见问题与应对

| 问题 | 现象 | 本质原因 | 解决方法 |
|------|------|----------|----------|
| 电机抖动 | 车一卡一卡的 | P 太大或没有滤波 | 降低 P，加滑动平均 |
| 停车过冲 | 过了目标位置还要滑一段 | D 太小 | 增加 D，或用斜坡减速 |
| 转向跑偏 | 一直往一边偏 | 左右轮机械特性不同 | 单独调每个轮的 PID 参数 |
| 编码器跳数 | 速度显示乱跳 | 接线松动或 EMI 干扰 | 缩短线，加磁环或滤波 |
| 启动冲击 | 起步时车往前一窜 | 目标速度突变 | 加斜坡启动 (Ramp) |
