## 3. 舵机

> **一句话概括**：让机械臂"听指令"转动到指定角度的装置

### 3.1 先理解：什么是舵机？

想象你的**手臂**：
- 有肌肉（电机）
- 有骨骼（结构）
- 有神经（控制信号）

当你大脑发出"手臂抬到 90 度"的指令时：
1. 神经把信号传到肌肉
2. 肌肉收缩/舒张
3. 骨骼带动手臂转动
4. **同时**：肌肉会一直调整，保持在这个角度

**舵机**就是一个自带这些功能的**集成件**：
- 里面有微型电机 + 减速齿轮 + 控制电路 + 电位器（检测角度）

### 3.2 两种舵机的区别

#### 3.2.1 PWM 舵机（MG996R、MG90S 等）

**控制方式**：像军训喊口号

| 口令 | PWM 高电平时间 | 角度 |
|------|----------------|------|
| 稍息 | 0.5ms | 0° |
| 立正 | 1.5ms | 90° |
| 向右看齐 | 2.0ms | 180° |

**原理**：
1. 控制器发出 PWM 信号
2. 舵机内部的比较器量"高电平有多长"
3. 根据时间控制电机转动
4. 电位器实时检测角度，接近目标时减速，到位停止

**代码**：
```c
// 角度转 PWM 比较值
void Servo_SetAngle(uint8_t angle)
{
    // 公式：脉宽 = 500 + angle * (2000/180)
    // 0°   → 500us
    // 90°  → 1500us
    // 180° → 2500us
    
    uint16_t pulse_us = 500 + (angle * 2000 / 180);
    
    // STM32 定时器计算
    // 定时器频率 = 84MHz / 84 = 1MHz
    // 周期 = 1/1MHz = 1us
    // 但 PWM 周期是 20ms = 20000us
    // 所以 Period = 20000
    
    uint16_t cmp = pulse_us * (SystemCoreClock / 1000000) / 20;
    __HAL_TIM_SET_COMPARE(&htim12, TIM_CHANNEL_1, cmp);
}
```

**优缺点**：
- ✅ 简单便宜
- ✅ 多路舵机不冲突
- ❌ 只能单方向控制（位置环在舵机内部）
- ❌ 精度一般

#### 3.2.2 串口舵机（Fashion Star、STS 系列）

**控制方式**：像用**对讲机**喊话

| 对讲机要素 | 串口舵机对应 |
|------------|--------------|
| 喊"3号" | 舵机 ID |
| "转到 2000" | 位置指令 |
| "慢慢转" | 速度指令 |
| "收到请回复" | 校验和 |

**通信格式**：

```
┌──────┬────┬─────┬──────┬─────────────────┬────────┐
│ 包头  │ ID │长度 │ 指令 │ 数据            │ 校验和 │
│FF FF  │ xx │ 09  │ 03   │ 2A xx xx xx xx  │   xx   │
└──────┴────┴─────┴──────┴─────────────────┴────────┘
```

**发送位置指令**：
```c
void Servo_SendPositionCommand(uint8_t id, uint16_t position)
{
    uint8_t command[13];
    
    // 1. 包头（告诉对方开始接收）
    command[0] = 0xFF;
    command[1] = 0xFF;
    
    // 2. 舵机 ID（像喊"3号舵机"）
    command[2] = id;
    
    // 3. 数据长度（后面有多少数据）
    command[3] = 0x09;  // 9 字节
    
    // 4. 指令（要干什么）
    command[4] = 0x03;  // 写
    
    // 5. 地址（写到舵机哪个寄存器）
    command[5] = 0x2A;  // 目标位置寄存器
    
    // 6. 位置数据（转到这里）
    command[6] = (uint8_t)(position & 0xFF);        // 低字节
    command[7] = (uint8_t)((position >> 8) & 0xFF); // 高字节
    
    // 7. 速度（慢慢转）
    command[8] = 0x00;  // 速度低字节
    command[9] = 0x00;  // 速度高字节
    
    // 8. 扩展参数（加速度）
    command[10] = 0xC4;
    command[11] = 0x09;
    
    // 9. 校验和（确认对方听懂）
    // 计算方式：把 ID 到最后一个数据加起来，然后取反
    uint8_t checksum = 0;
    for (int i = 2; i < 12; i++) checksum += command[i];
    command[12] = ~checksum;  // 取反
    
    // 10. 发送！
    HAL_UART_Transmit(&huart5, command, 13, HAL_MAX_DELAY);
}
```

**优缺点**：
- ✅ 精度高（12 位 = 4096 档）
- ✅ 可控制速度、加速度
- ✅ 可读取位置、速度、温度
- ❌ 需要更多接线（共用总线）
- ❌ 协议复杂

### 3.3 进阶功能：带加速度的速度控制

普通转舵机：直接转到目标 → **咔嚓一声**

带加速度：慢慢加速 → 匀速 → 慢慢减速 → **平滑**

```c
void Servo_SendPositionAccelSpeedCommand(
    UART_HandleTypeDef *huart,  // 用哪个串口
    uint8_t id,                  // 舵机 ID
    uint16_t acceleration,       // 加速度 (0-255)
    int position,                // 目标位置 (0-4095)
    uint16_t speed)             // 最大速度 (0-3400)
{
    uint8_t command[14];
    
    command[0] = 0xFF; command[1] = 0xFF;
    command[2] = id;
    command[3] = 0x0A;  // 长度变成 10（多了加速度）
    command[4] = 0x03;  // 写
    command[5] = 0x29;  // 地址变成 0x29（加速度寄存器）
    
    // 数据区
    command[6] = acceleration;                      // 加速度
    command[7] = (uint8_t)(position & 0xFF);       // 位置低
    command[8] = (uint8_t)(position >> 8);         // 位置高
    command[9] = 0x00; command[10] = 0x00;        // 时间（不用）
    command[11] = (uint8_t)(speed & 0xFF);          // 速度低
    command[12] = (uint8_t)(speed >> 8);            // 速度高
    
    // 校验和
    uint8_t checksum = 0;
    for (int i = 2; i < 13; i++) checksum += command[i];
    command[13] = ~checksum;
    
    HAL_UART_Transmit(huart, command, 14, HAL_MAX_DELAY);
}
```

### 3.4 环形缓冲区——不会丢失的数据

**问题**：串口一个个字节接收，如果处理太慢会丢失！

**解决**：就像你**录音**：
- 麦克风一直录（DMA 自动写入）
- 你一段一段听（CPU 空闲时处理）
- 录满了？**覆盖最早的**（环形）

```c
typedef struct {
    uint8_t *buffer;    // 存数据的数组
    uint32_t size;      // 总大小
    uint32_t head;      // 写指针（放哪里）
    uint32_t tail;      // 读指针（从哪里取）
    uint32_t count;     // 当前有多少数据
} RingBufferTypeDef;

// 写入
void RingBuffer_WriteByte(RingBufferTypeDef *rb, uint8_t data)
{
    rb->buffer[rb->head++] = data;       // 写入
    if (rb->head >= rb->size)            // 循环
        rb->head = 0;
    rb->count++;                         // 计数 +1
}

// 读取
uint8_t RingBuffer_ReadByte(RingBufferTypeDef *rb)
{
    uint8_t data = rb->buffer[rb->tail++];
    if (rb->tail >= rb->size)
        rb->tail = 0;
    rb->count--;
    return data;
}
```

### 3.5 实际应用：机械臂控制

#### 3.5.1 硬件连接

| 舵机 | 功能 | 控制方式 |
|------|------|----------|
| 底座舵机 | 左右旋转 | UART5 |
| 大臂舵机 | 抬降 | PWM |
| 小臂舵机 | 俯仰 | PWM |
| 爪子舵机 | 开合 | PWM |

#### 3.5.2 控制流程

```
主循环：
  1. 接收任务指令（如"抓取 A 位置"）
  2. 计算每个舵机目标角度
  3. 按顺序发送指令（底座→大臂→小臂→爪子）
  4. 等待转动到位
  5. 执行抓取/放置
```

### 3.6 常见问题与应对

| 问题 | 现象 | 本质原因 | 解决方法 |
|------|------|----------|----------|
| 舵机抖动 | 一直嗡嗡响 | PWM 频率不对或信号干扰 | 确认 50Hz，检查杜邦线 |
| 舵机不转 | 叫了不动 | 协议错误或供电不足 | 校验和重算，增加供电 |
| 角度不准 | 转不到指定位置 | 脉宽映射有偏差 | 实际测量调整公式 |
| 转动卡顿 | 一卡一卡的 | 串口数据丢失 | 用 DMA + 缓冲区 |
| 发热严重 | 烫手 | 负载太大或堵转 | 减轻负载，加散热 |
