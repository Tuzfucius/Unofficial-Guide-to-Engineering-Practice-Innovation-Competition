### 4.2 OpenMV RPC 通信

#### 4.2.1 通信协议

OpenMV 使用 RPC (Remote Procedure Call) 协议进行通信：

```
[Header]     [Payload Size] [Function Name] [Args...] [Checksum]
0x00 0x00    xx xx          xxx...         xx...   xx
```

    RPC_STATE_IDLE,

static volatile rpc_state_t rpc_state = RPC_STATE_IDLE;

#### 4.2.3 数据接收 (DMA + IDLE 中断)

```c
void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart, uint16_t Size)
{
    if (huart->Instance == USART6)
    {
        // 1. 停止 DMA
        HAL_UART_DMAStop(&huart6);
        
        // 2. 拷贝数据到应用缓冲区
        memcpy(openmv_process_buffer, openmv_dma_rx_buffer, Size);
        
        // 3. 设置数据就绪标志
        openmv_data_length = Size;
        openmv_data_ready = true;
        
        // 4. 重新启动 DMA 接收
        HAL_UARTEx_ReceiveToIdle_DMA(&huart6, openmv_dma_rx_buffer, OPENMV_RX_BUFFER_SIZE);
    }
}
```

**精华解读**：
- 使用 **DMA + UART IDLE 中断**实现不定长数据接收
- 当检测到总线空闲（一帧数据接收完成）时触发中断

#### 4.2.4 视觉数据解析

```c
// 全局变量存储解析结果
char target_color = ' ';
int target_x = -1;
int target_y = -1;
int target_angle = -1;

void parse_openmv_data(void)
{
    // 格式: "c,x,y,a\n" e.g., "R,50,30,45\n"
    int items_parsed = sscanf((const char *)openmv_process_buffer, 
                              "%c,%d,%d,%d",
                              &target_color, &target_x, &target_y, &target_angle);
    
    if (items_parsed == 4)
    {
        printf("Parse OK -> Color: %c, X: %d, Y: %d, Angle: %d\r\n",
               target_color, target_x, target_y, target_angle);
    }
    else
    {
        target_color = ' ';
        target_x = target_y = target_angle = -1;
    }
    
    memset(openmv_process_buffer, 0, sizeof(openmv_process_buffer));
}
```

### 4.4 主循环中的视觉处理

### 4.5 常见问题与改进建议

| 问题 | 原因 | 改进建议 |
|------|------|----------|
| 视觉数据丢失 | DMA 缓冲区太小 | 增大缓冲区，或使用更可靠的协议 |
| 解析失败 | 字符串格式不匹配 | 确保 OpenMV 发送格式稳定，使用校验位 |
| 响应延迟 | 主循环阻塞 | 使用中断 + 标志位机制 |
| 目标丢失抖动 | 阈值设置不当 | 调整色块检测参数，增加滤波 |
