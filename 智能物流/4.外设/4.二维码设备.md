### 4.1 系统架构

#### 4.1.1 硬件组成

```
┌─────────────┐    UART (DMA)    ┌─────────────┐
│   OpenMV    │ ←──────────────→ │    STM32    │
│ (摄像+算法)  │   "R,50,30,45"   │ (决策控制)   │
└─────────────┘                   └─────────────┘
```

**OpenMT**：自带摄像头的 Python 可编程模块
- 内置色块检测、QR 码识别、神经网络
- 通过 UART 发送检测结果

**STM32**：主控制器
- 接收视觉数据
- 执行运动控制决策

#### 4.1.2 数据流向

```
OpenMV 检测流程：
摄像头 → 图像处理 → 目标识别 → 格式化 → UART 发送

STM32 接收流程：
DMA 接收 → 空闲中断 → 数据拷贝 → 解析 → 控制决策
```

### 4.2 OpenMV 通信协议

#### 4.2.1 数据格式

OpenMV 通过 UART 发送字符串格式数据：

```
"颜色,X,Y,角度\n"

示例："R,50,30,45\n"
```

**字段说明**：

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| 颜色 | char | R/G/B | 目标颜色 |
| X | int | 0-127 | 图像横坐标 |
| Y | int | 0-63 | 图像纵坐标 |
| 角度 | int | 0-180 | 检测角度 |

#### 4.2.2 坐标系

```
(0,0) ──────────────→ X (列)
  │
  │
  │
  ↓
  Y (行)
```

图像左上角为原点，X 向右增加，Y 向下增加。

### 4.3 DMA + 空闲中断

#### 4.3.1 DMA 原理

| 方式 | CPU 占用 | 效率 | 适用场景 |
|------|----------|------|----------|
| 轮询 | 高 | 低 | 简单场景 |
| 中断 | 中 | 中 | 中等速度 |
| DMA | 低 | 高 | 高速数据 |

**DMA 优点**：
- 硬件自动传输数据
- CPU 可并行处理其他任务
- 减少中断次数

#### 4.3.2 空闲中断原理

串口数据无固定长度，通过"空闲检测"判断一帧结束：

- 串口接收过程中，IDLE 引脚保持低电平
- 一帧结束 → 总线空闲 → IDLE 中断触发

```c
/**
 * @brief DMA 接收配置
 * @note 使用 HAL 库函数启动 DMA + 空闲中断
 */
HAL_UARTEx_ReceiveToIdle_DMA(&huart6, rx_buffer, BUFFER_SIZE);

/**
 * @brief UART 接收事件回调
 * @param huart: UART 句柄
 * @param Size: 本次接收的数据长度
 * 
 * 触发条件：
 * 1. DMA 缓冲区满
 * 2. 空闲中断检测到总线空闲（一帧结束）
 */
void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart, uint16_t Size)
{
    if (huart->Instance == USART6)  // OpenMV 使用 USART6
    {
        // 1. 停止 DMA（防止数据竞争）
        HAL_UART_DMAStop(&huart6);

        // 2. 拷贝数据到应用缓冲区
        memcpy(process_buffer, rx_buffer, Size);

        // 3. 设置数据就绪标志
        data_ready = true;
        data_length = Size;

        // 4. 重新启动 DMA，准备下一帧
        HAL_UARTEx_ReceiveToIdle_DMA(&huart6, rx_buffer, BUFFER_SIZE);
    }
}
```

### 4.4 数据解析

#### 4.4.1 sscanf 解析

```c
/**
 * @brief 解析 OpenMV 数据
 * @note 格式："R,50,30,45"
 * 
 * sscanf 格式串说明：
 * %c  → 单个字符
 * %d  → 有符号整数
 * %s  → 字符串
 */
void parse_openmv_data(void)
{
    // 解析字符串
    int parsed = sscanf(
        (const char *)process_buffer,
        "%c,%d,%d,%d",
        &target_color,    // %c → 'R'
        &target_x,       // %d → 50
        &target_y,       // %d → 30
        &target_angle    // %d → 45
    );

    if (parsed == 4)  // 成功解析 4 个字段
    {
        // 解析成功，使用数据
    }
    else
    {
        // 解析失败，重置为无效值
        target_color = ' ';
        target_x = target_y = target_angle = -1;
    }

    // 清空缓冲区
    memset(process_buffer, 0, BUFFER_SIZE);
}
```

#### 4.4.2 数据有效性验证

```c
/**
 * @brief 验证数据有效性
 * @return true: 有效, false: 无效
 */
bool validate_data(void)
{
    // 检查颜色
    if (target_color != 'R' && target_color != 'G' && target_color != 'B')
        return false;

    // 检查坐标范围
    if (target_x < 0 || target_x > 127)
        return false;
    if (target_y < 0 || target_y > 63)
        return false;

    // 检查角度范围
    if (target_angle < 0 || target_angle > 180)
        return false;

    return true;
}
```

### 4.5 控制指令发送

#### 4.5.1 补光灯控制

```c
/**
 * @brief 设置 OpenMV 补光灯亮度
 * @param brightness: 亮度 (0-100)
 * 
 * 指令格式："L,100\n"
 */
void set_openmv_light(uint8_t brightness)
{
    char cmd[10];
    sprintf(cmd, "L,%d\n", brightness);

    HAL_UART_Transmit(&huart6,
                      (uint8_t *)cmd,
                      strlen(cmd),
                      HAL_MAX_DELAY);
}
```

void toggle_openmv_mode(void)

}

### 4.6 主循环数据处理

```c
/**
 * @brief 主循环中的视觉处理
 * @note 非阻塞轮询方式
 */
int main(void)
{
    // 初始化...
    center_x = 78;
    center_y = 30;

    while (1)
    {
        // 1. 检查数据就绪标志
        if (openmv_data_ready)
        {
            // 2. 解析数据
            parse_openmv_data();
            openmv_data_ready = false;
        }

        // 3. 目标跟踪逻辑
        if (target_color == 'R')  // 检测到红色目标
        {
            // 计算偏差
            int x_error = target_x - center_x;
            int y_error = target_y - center_y;

            // X 方向控制
            if (x_error < -10)
                Car_TurnLeft(300);   // 偏左，左转
            else if (x_error > 10)
                Car_TurnRight(300);  // 偏右，右转
            else
                Car_Go(500);         // 居中，前进
        }

        // 4. 急停处理
        Car_Process_Stop_Request();
    }
}
```

### 4.9 常见问题分析

| 问题 | 本质原因 | 解决方案 |
|------|----------|----------|
| 无数据接收 | 接线错误或波特率不匹配 | 检查 RX/TX 交叉连接 |
| 解析失败 | 字符串格式不一致 | 统一 OpenMV 输出格式 |
| 响应延迟 | 主循环阻塞 | 中断+标志位机制 |
| 目标丢失 | 检测阈值不当 | 调整阈值，增加预测 |
| 误识别 | 光照干扰 | 滤光片或阈值自适应 |
