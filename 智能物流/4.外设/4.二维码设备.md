## 4. 二维码设备 & 视觉识别

本节阐述智能物流小车视觉系统的设计与实现，包括 OpenMV 视觉模块通信、二维码识别及主控系统数据处理流程。

### 4.1 系统架构

#### 4.1.1 视觉系统组成

```
┌─────────────┐    UART (DMA)    ┌─────────────┐
│   OpenMV    │ ←──────────────→ │    STM32    │
│ (图像采集)   │    "x,y,angle"   │ (决策控制)   │
└─────────────┘                   └─────────────┘
```

**OpenMV**：自带摄像头的 Python 可编程模块，内置色块检测、QR 码识别、神经网络等算法。

**STM32**：主控制器，接收视觉数据并执行运动控制。

#### 4.1.2 数据流向

```
OpenMV          STM32                    应用层
────────────────────────────────────────────────────
检测色块 → 格式化字符串 → UART发送 → DMA接收 → 解析 → 控制决策
```

### 4.2 OpenMV 通信协议

#### 4.2.1 数据格式

OpenMV 通过 UART 发送字符串格式数据：

```
"颜色,X,Y,角度\n"

示例："R,50,30,45\n"
```

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| 颜色 | char | R/G/B | 目标颜色 |
| X | int | 0-127 | 图像横坐标 |
| Y | int | 0-63 | 图像纵坐标 |
| 角度 | int | 0-180 | 检测角度 |

#### 4.2.2 坐标系

```
(0,0) ──────────────→ X (列)
  │
  │
  │
  ↓
  Y (行)
```

### 4.3 DMA + 空闲中断接收

#### 4.3.1 为什么用 DMA？

传统 CPU 轮询方式效率低下，DMA 实现硬件级数据搬运：

| 方式 | CPU 占用 | 效率 |
|------|----------|------|
| 轮询 | 高 | 低 |
| 中断 | 中 | 中 |
| DMA | 低 | 高 |

#### 4.3.2 空闲中断原理

串口数据无固定长度，通过"空闲检测"判断一帧结束：

- 串口接收过程中，IDLE 引脚保持低电平
- 一帧结束 → 总线空闲 → IDLE 中断触发

```c
// DMA 接收配置
HAL_UARTEx_ReceiveToIdle_DMA(&huart6, rx_buffer, BUFFER_SIZE);

// 中断回调
void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart, uint16_t Size)
{
    if (huart->Instance == USART6)
    {
        // 1. 停止 DMA
        HAL_UART_DMAStop(&huart6);

        // 2. 拷贝数据
        memcpy(process_buffer, rx_buffer, Size);

        // 3. 设置标志
        data_ready = true;
        data_length = Size;

        // 4. 重启 DMA
        HAL_UARTEx_ReceiveToIdle_DMA(&huart6, rx_buffer, BUFFER_SIZE);
    }
}
```

### 4.4 数据解析

#### 4.4.1 sscanf 解析

```c
void parse_openmv_data(void)
{
    // 格式："R,50,30,45"
    // %c = char, %d = int

    int parsed = sscanf(
        (const char *)process_buffer,
        "%c,%d,%d,%d",
        &target_color,
        &target_x,
        &target_y,
        &target_angle
    );

    if (parsed == 4)
    {
        // 解析成功
    }
    else
    {
        target_color = ' ';
        target_x = target_y = target_angle = -1;
    }

    memset(process_buffer, 0, BUFFER_SIZE);
}
```

#### 4.4.2 数据有效性验证

- 坐标范围检查：x ∈ [0, 127], y ∈ [0, 63]
- 颜色字符过滤：仅接受 R/G/B
- 连续无效帧计数：超过阈值触发报警

### 4.5 控制指令发送

#### 4.5.1 补光灯控制

```c
void set_openmv_light(uint8_t brightness)
{
    char cmd[10];
    sprintf(cmd, "L,%d\n", brightness);

    HAL_UART_Transmit(&huart6,
                      (uint8_t *)cmd,
                      strlen(cmd),
                      HAL_MAX_DELAY);
}
```

#### 4.5.2 模式切换

```c
void toggle_openmv_mode(void)
{
    const char *cmd = "M\n";
    HAL_UART_Transmit(&huart6, (uint8_t *)cmd, 2, HAL_MAX_DELAY);
}
```

### 4.6 主循环数据处理

```c
int main(void)
{
    // 初始化...
    center_x = 78;
    center_y = 30;

    while (1)
    {
        // 1. 检查数据就绪
        if (openmv_data_ready)
        {
            parse_openmv_data();
            openmv_data_ready = false;
        }

        // 2. 目标跟踪逻辑
        if (target_color == 'R')
        {
            if (target_x < center_x - 10)
                Car_TurnLeft(300);
            else if (target_x > center_x + 10)
                Car_TurnRight(300);
            else
                Car_Go(500);
        }

        // 3. 急停处理
        Car_Process_Stop_Request();
    }
}
```

### 4.9 常见问题分析

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无数据接收 | 接线错误或波特率不匹配 | 检查 RX/TX 交叉连接 |
| 解析失败 | 字符串格式不一致 | 统一 OpenMV 输出格式 |
| 响应延迟 | 主循环阻塞 | 中断+标志位机制 |
| 目标丢失 | 检测阈值不当 | 调整阈值，增加预测 |
| 误识别 | 光照干扰 | 滤光片或阈值自适应 |
