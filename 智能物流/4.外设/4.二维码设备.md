## 4. 二维码设备 & 视觉识别

> **一句话概括**：让小车"看见"东西——识别二维码和色块

### 4.1 想象一下

你的眼睛看到红灯 → 大脑判断"不能走" → 脚踩刹车

视觉系统就是小车的**眼睛和简单的大脑**：
- **摄像头**（OpenMV/二维码模块）→ 眼睛
- **通信协议** → 神经系统，把看到的说给大脑（STM32）听
- **STM32** → 根据看到的做决策

---

### 4.2 OpenMV——自带大脑的摄像头

#### 什么是 OpenMV？

想象一个**树莓派 + 摄像头**的组合：
- 能识别色块、线条、二维码
- 内置 Python 解释器，可以自己跑程序
- 通过串口把结果告诉 STM32

#### 数据怎么传？

格式：`"颜色,X坐标,Y坐标,角度\n"`

例如：`"R,50,30,45\n"` 表示：
- R = 红色
- X = 50（图像中心右侧 50 像素）
- Y = 30（图像中心下方 30 像素）
- 角度 = 45°

#### STM32 怎么接收？（DMA + 空闲中断）

这就像**专人 + 举牌子**：
- **DMA**：专人负责接收数据，不用 CPU 一直盯着
- **空闲中断**：当举牌子的人停下来（一帧结束），CPU 过来处理

```c
// DMA 接收完成中断
void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart, uint16_t Size)
{
    if (huart->Instance == USART6)  // OpenMV 用的是 USART6
    {
        // 1. 把数据从 DMA 缓冲区拷贝出来
        memcpy(openmv_process_buffer, openmv_dma_rx_buffer, Size);
        
        // 2. 标记"数据到了！"
        openmv_data_ready = true;
        
        // 3. 继续等待下一帧
        HAL_UARTEx_ReceiveToIdle_DMA(&huart6, openmv_dma_rx_buffer, OPENMV_RX_BUFFER_SIZE);
    }
}
```

### 4.3 解析数据——看懂它在说什么

```c
void parse_openmv_data(void)
{
    // 格式: "R,50,30,45"
    int items = sscanf((const char *)openmv_process_buffer, 
                       "%c,%d,%d,%d",
                       &target_color, &target_x, &target_y, &target_angle);
    
    if (items == 4)  // 成功解析 4 个数据
    {
        printf("看到 %c 色块，位置(%d,%d)，角度%d\n",
               target_color, target_x, target_y, target_angle);
    }
    else  // 解析失败
    {
        target_color = ' ';
        target_x = target_y = target_angle = -1;
    }
}
```

### 4.4 控制 OpenMV——发送指令

```c
// 让 OpenMV 打开补光灯
void set_openmv_light(uint8_t brightness)
{
    char cmd[10];
    sprintf(cmd, "L,%d\n", brightness);  // "L,100\n"
    
    HAL_UART_Transmit(&huart6, (uint8_t *)cmd, strlen(cmd), HAL_MAX_DELAY);
}
```

### 4.5 二维码识别（USART2）

流程和 OpenMV 类似，但数据格式不同：

```c
void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart, uint16_t Size)
{
    if (huart->Instance == USART2)  // 二维码模块
    {
        qr_uart_rx_event_callback(huart, Size);
```

### 4.6 主循环中的视觉处理

```c
int main(void)
{
    // ... 初始化 ...
    
    while (1)
    {
        // 1. 看看有没有新数据
        if (openmv_data_ready)
        {
            // 2. 解析数据
            parse_openmv_data();
            openmv_data_ready = false;
        }
        
        // 3. 根据看到的东西做反应
        if (target_color == 'R')  // 看到红色
        {
            Car_TurnLeft(500);    // 左转
        }
        
        Car_Process_Stop_Request();  // 处理急停
    }
}
```

### 4.7 常见问题与应对

| 问题 | 就像... | 解决方法 |
|------|---------|----------|
| 收不到数据 | 耳朵聋了 | 检查串口接线、波特率（115200） |
| 解析错误 | 听岔了 | 检查 OpenMV 发送格式是否稳定 |
| 响应慢 | 反应迟钝 | 用中断 + 标志位，不要阻塞等待 |
| 目标丢失 | 跟丢了 | 调整色块检测阈值，增加滤波 |
