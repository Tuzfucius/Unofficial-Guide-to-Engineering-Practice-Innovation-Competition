## 4. 二维码设备 & 视觉识别

> **一句话概括**：让小车"看见"东西——识别二维码和色块

### 4.1 先理解：为什么需要视觉？

想象你闭着眼走路：
- 只能靠摸
- 撞墙还不知道
- 走不远

**加上眼睛**：
- 看到障碍物 → 绕开
- 看到路标 → 知道往哪走
- 看到目标 → 抓取

小车也是一样的：只有传感器（编码器、陀螺仪）= 闭着眼走；加上视觉 = 睁开眼睛

### 4.2 视觉系统的组成

```
┌─────────────┐    串口     ┌─────────────┐
│   OpenMV    │ ──────────→ │    STM32    │
│ (摄像+处理)  │   "我看到了" │  (做决策)   │
└─────────────┘              └─────────────┘
```

**OpenMV** = 摄像头 + Python 控制器
- 自带镜头和图像处理算法
- 可以跑 Python 程序
- 通过串口把结果告诉 STM32

**STM32** = 小车的大脑
- 接收 OpenMV 的数据
- 根据看到的内容做决策

### 4.3 OpenMV 通信原理

#### 4.3.1 数据格式

OpenMV 用**字符串**发数据，像发微信：

```
"颜色,X坐标,Y坐标,角度\n"

例子："R,50,30,45\n"
```

| 字段 | 含义 | 范围 |
|------|------|------|
| R | 颜色 (R/G/B) | R/G/B |
| 50 | X 坐标 | 0-127 |
| 30 | Y 坐标 | 0-63 |
| 45 | 角度 | 0-180 |

**图像坐标系**：

```
(0,0) ──────────────→ X
  │
  │
  │
  │
  ↓
  Y
```

### 4.4 STM32 怎么接收？—— DMA + 空闲中断

#### 4.4.1 为什么需要 DMA？

想象你在**收快递**：
- **不用 DMA**：你一直站在门口，每来一个快递亲自接 → **累死**
- **用 DMA**：雇个快递员（DMA），收到一堆再叫你 → **空闲时处理**

**DMA** = Direct Memory Access，直接内存访问
- 硬件自动把数据搬到内存
- CPU 不用一直盯着

#### 4.4.2 为什么要用"空闲中断"？

问题：不知道一帧数据有多长！

- 微信消息：长度在消息里
- 串口数据：没有长度字段

**解决**：等"空闲"（一段时间没新数据 = 一帧结束）

```c
// DMA 接收完成 + 空闲中断触发
void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart, uint16_t Size)
{
    if (huart->Instance == USART6)  // OpenMV 用 USART6
    {
        // 1. 停一下 DMA（防止正在写的时候又被覆盖）
        HAL_UART_DMAStop(&huart6);
        
        // 2. 把数据从 DMA 缓冲区拷贝出来（到应用缓冲区）
        memcpy(openmv_process_buffer, openmv_dma_rx_buffer, Size);
        
        // 3. 标记"数据到了！"
        openmv_data_ready = true;
        openmv_data_length = Size;
        
        // 4. 继续等待下一帧
        HAL_UARTEx_ReceiveToIdle_DMA(&huart6, 
                                      openmv_dma_rx_buffer, 
                                      OPENMV_RX_BUFFER_SIZE);
    }
}
```

### 4.5 解析数据——看懂它在说什么

```c
void parse_openmv_data(void)
{
    // 用 sscanf 解析字符串
    // 格式: "R,50,30,45"
    // %c = 字符, %d = 整数
    
    int items = sscanf(
        (const char *)openmv_process_buffer,
        "%c,%d,%d,%d",
        &target_color,   // %c → 'R'
        &target_x,       // %d → 50
        &target_y,       // %d → 30
        &target_angle    // %d → 45
    );
    
    if (items == 4)  // 成功解析 4 个数据
    {
        printf("看到 %c 色块，位置(%d,%d)，角度=%d\n",
               target_color, target_x, target_y, target_angle);
    }
    else
    {
        // 解析失败，重置
        target_color = ' ';
        target_x = target_y = target_angle = -1;
    }
}
```

### 4.6 控制 OpenMV——发送指令

#### 4.6.1 打开补光灯

```c
void set_openmv_light(uint8_t brightness)
{
    char cmd[10];
    // 格式: "L,100\n"
    sprintf(cmd, "L,%d\n", brightness);
    
    HAL_UART_Transmit(&huart6, 
                      (uint8_t *)cmd, 
                      strlen(cmd), 
                      HAL_MAX_DELAY);
}
```

#### 4.6.2 切换模式

```c
void toggle_openmv_mode(void)
{
    const char *cmd = "M\n";
    HAL_UART_Transmit(&huart6, (uint8_t *)cmd, 2, HAL_MAX_DELAY);
}
```

### 4.7 主循环中的视觉处理

```c
int main(void)
{
    // ... 初始化 ...
    
    // 设置 OpenMV 中心点
    center_x = 78;
    center_y = 30;
    
    while (1)
    {
        // 1. 看看有没有新数据
        if (openmv_data_ready)
        {
            // 2. 解析数据
            parse_openmv_data();
            openmv_data_ready = false;
        }
        
        // 3. 根据看到的内容做反应
        if (target_color == 'R')  // 看到红色
        {
            // 偏左 → 往左转
            if (target_x < center_x - 10)
                Car_TurnLeft(300);
            // 偏右 → 往右转
            else if (target_x > center_x + 10)
                Car_TurnRight(300);
            // 在中心 → 直行
            else
                Car_Go(500);
        }
        
        // 4. 处理急停
        Car_Process_Stop_Request();
    }
}
```

### 4.8 二维码识别

#### 4.8.1 和 OpenMV 的区别

| 特性 | OpenMV | 二维码模块 |
|------|--------|------------|
| 功能 | 色块、线条、形状识别 | 专用 QR 码解码 |
| 优点 | 功能多、可编程 | 解码快、准 |
| 缺点 | 复杂场景慢 | 只能二维码 |

### 4.9 常见问题与应对

| 问题 | 现象 | 本质原因 | 解决方法 |
|------|------|----------|----------|
| 收不到数据 | 一直收不到 | 接线错误/波特率不对 | 检查 RX/TX 是否接反 |
| 解析错误 | 数据全是错的 | 字符串格式不匹配 | 确认 OpenMV 发送格式 |
| 响应延迟 | 看到东西但不动 | 主循环阻塞 | 用中断 + 标志位机制 |
| 目标丢失 | 跟丢目标 | 检测阈值不合适 | 调整色块阈值，加跟踪 |
| 误识别 | 认错颜色 | 光照影响 | 加滤光片或调阈值 |
